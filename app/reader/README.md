# YUI 阅读器应用

这是一个基于 YUI 框架开发的电子书阅读器应用，支持从外部文件中读取内容并进行阅读。

## 功能特点

- 📚 支持多本书籍的阅读
- 📖 自动分页功能
- 🌙 日间/夜间主题切换
- 🔤 字体大小调整（12px-24px）
- 📊 阅读进度显示
- 🔄 章节导航
- 📁 直接从文件系统读取书籍内容

## 文件结构

```
reader/
├── reader.json          # 应用UI定义
├── reader.js           # 应用逻辑
├── books/              # 书籍文件夹
│   ├── book1.txt       # 第一章：开始
│   ├── book2.txt       # 第二章：发展
│   └── book3.txt       # 第三章：高潮
└── README.md           # 本文件
```

## 使用方法

1. 在 YUI 框架中加载 `reader/reader.json` 文件
2. 应用会自动初始化并从文件加载第一章内容
3. 使用工具栏的下拉框切换书籍
4. 使用底部导航按钮翻页
5. 点击主题切换按钮切换日间/夜间模式
6. 使用字体大小按钮调整字体大小

## 技术实现

### 文件读取方式

我们通过以下方式实现了从文件系统读取内容：

1. **添加文件读取 API**: 在 `js_common.c` 中添加了 `js_module_read_file` 函数，用于读取文件内容。

2. **暴露给 JavaScript**: 在 `yui_stdlib.c` 中添加了 `js_yui_read_file` 函数，将文件读取功能暴露给 JavaScript，通过 `YUI.readFile(path)` 调用。

3. **JavaScript 实现**: 在 `reader.js` 中使用 `YUI.readFile(bookPath)` 直接从文件系统读取书籍内容。

### 事件处理

应用中的事件通过以下方式处理：

1. **初始化**: 在 `reader.json` 中通过 `"initialize": "initialize"` 指定初始化函数。

2. **UI 事件**: 按钮点击、选择框变化等事件通过 JSON 中的 `onClick` 和 `onChange` 属性绑定到 JavaScript 函数。

3. **函数导出**: JavaScript 函数通过 `global` 对象导出，使 YUI 能够调用它们。

### 主题切换

1. **主题定义**: 主题定义在 `reader.json` 的 `themes` 部分。

2. **主题切换**: 使用 YUI 的主题 API (`YUI.themeLoad`, `YUI.themeSetCurrent`, `YUI.themeApplyToTree`) 切换主题。

## 扩展指南

### 添加新书

1. 在 `books/` 文件夹中添加新的 TXT 文件。
2. 在 `reader.json` 中的 `bookSelector` 组件中添加新书的选项。

### 自定义主题

1. 在 `reader.json` 的 `themes` 部分添加新主题。
2. 在 JavaScript 中修改 `toggleTheme` 函数以支持新主题。

### 添加新功能

1. 在 `reader.js` 中添加新的函数。
2. 通过 `global` 对象导出函数，使其可供 YUI 调用。
3. 在 `reader.json` 中添加相应的 UI 组件并绑定事件。

## 技术细节

### 文件读取 API

我们添加的 `YUI.readFile(path)` API 具有以下特点：

- **同步读取**: 当前实现是同步的，会阻塞 UI 直到文件读取完成。
- **路径处理**: 路径是相对于应用程序工作目录的。
- **错误处理**: 如果文件不存在或无法读取，函数返回 `undefined`。

### 分页算法

阅读器使用智能分页算法：

1. **页面大小**: 每页最多 1000 个字符。
2. **智能分割**: 尽量在句号、感叹号或问号处分割页面。
3. **备用分割**: 如果找不到标点符号，则在空格处分割。
4. **最后手段**: 如果以上都失败，则强制分割。

## 注意事项

1. **文件编码**: 目前支持 UTF-8 编码的文本文件。

2. **文件路径**: 书籍文件必须放在 `app/reader/books/` 目录下。

3. **阅读进度**: 当前阅读进度仅保存在内存中，应用重启后会丢失。如果需要持久化存储，可以考虑使用 YUI 提供的存储 API（如果有）。

4. **性能**: 对于大文件，可能会在首次加载时有一些延迟，因为需要一次性读取整个文件并进行分页。

## 故障排除

1. **内容不显示**: 
   - 检查文件路径是否正确
   - 确保文件存在且可读
   - 检查浏览器控制台是否有错误信息

2. **事件不响应**: 
   - 确保事件函数已通过 `global` 对象导出
   - 检查函数名与 JSON 中的事件绑定是否匹配

3. **主题切换不生效**: 
   - 检查主题文件路径和主题名称是否正确
   - 确保 YUI 的主题 API 调用正确

4. **文件读取失败**: 
   - 检查 YUI.readFile API 是否可用
   - 确认文件路径是否正确
   - 检查文件权限是否允许读取
